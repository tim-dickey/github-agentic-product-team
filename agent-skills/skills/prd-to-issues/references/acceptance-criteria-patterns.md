# Acceptance criteria patterns

Good acceptance criteria are:
- observable
- specific
- tied to user or system behavior
- free from vague language

## Better patterns

Use:
- "When the user submits the form with valid inputs, the record is saved and a success message appears."
- "If the API returns 401, the UI redirects the user to sign in again."
- "The audit log stores the actor, action, timestamp, and target record ID."

Avoid:
- "Works correctly"
- "Handles errors well"
- "UI is intuitive"
- "System is robust"
