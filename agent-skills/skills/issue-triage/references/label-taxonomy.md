# Label taxonomy

Every new issue should receive:
- one `type:*` label
- one `area:*` label
- one `severity:*` label

Do not apply multiple labels from the same family unless a human explicitly requests it.

## Type meanings

- `type:bug` - existing behavior is wrong or broken
- `type:feature` - request for new capability
- `type:support` - help request, usage confusion, setup problem
- `type:docs` - documentation-only issue
- `type:duplicate` - issue is already tracked elsewhere

## Area meanings

- `area:web` - customer-facing UI
- `area:admin` - internal admin UI
- `area:api` - backend services or endpoints
- `area:auth` - login, sessions, permissions, SSO
- `area:billing` - payments, invoices, subscriptions, refunds
- `area:notifications` - email, SMS, push, webhooks
- `area:analytics` - tracking, dashboards, event flows
- `area:docs` - documentation content or information architecture
